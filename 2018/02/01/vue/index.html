HTML 中 is 属性

There is no is attribute in HTML.
It is a proposed extension that appears in the Custom Elements specification (which evolved from the Polymer spec mentioned below).
It allows you to say that an existing, standard element is really a custom element.

vue 标签及说明

  • v-on 对于静态渲染，可以提高效率
  • v-html
  • v-bind:xxx="表达式"，缩写是 :xxx="表达式"
  • v-if, v-else,v-else-fi
  • v-on:click="xxx"，缩写是 @click="xxx"
  • v-model="变量"，用于表单输入双向绑定
  • v-show 不同于 v-if，控制 display属性，元素实际是存在的
  • v-for="item in items"

v-for

v-for=”item in items”，也可以换成 v-for=”item of items”，这样更符合js的迭代器语法。
重温下：in 和 of 的区别：
in 是 key，可以用于迭代对象和普通object。对于一个列表，in 返回的对象是 string 类型的 0,1,2,3 …
of 才是迭代器中的对象，非迭代对象没有该方法

只要想要传入 js 的表达式，就必须要用 v-bind

vue 对于 class，style 的 特殊处理，v-bind

如果想要动态的调整 class，即用变量控制 class，则肯定是要用 v-bind
vue 提供了三种 class 的配置方式：

  1. :class=”{classA: true, classB: false}”，即对象语法。
  2. :class=”[‘classA’, ‘classB’]”，即数组语法。
  3. :class=”[{classA: true}, {classB: false}]”，混合语法

可以把当做是不可见的包裹块来用，可加上 v-if 这类的控制语句。

key

如果重新渲染后，控件的内容还在上面，可以加上 key 来解决。原因是为了提高性能，不会渲染它觉得是一样的控件。

html 不区分大小写！所以标签的 attr 都是用 - 分隔的名称

vue 最大的特色就是

比如 v-model 很好用，不用再像 react 用的 jsx 语法，自己来手动实现一个双向绑定功能了。另外它将控制语句全部写在标签中，而表达式{}部分只放数据。
对比 react 使用的 jsx 语法，控制语句只能写在表达式{}中，这样会显得有些乱吧…
其实，vue 也提供直接手写 render 函数的功能，我们只要在 render 函数里面写 jsx 语法就行了（前提装上 babel-loader），这样就接近 react 的习惯了，但是就不能用那些 vue 替我们封装好的 v-xxx 标签。
